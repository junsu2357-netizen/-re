CHAPTER A 클라우드 기초 개념
🏢 클라우드 (Cloud) CLOUD ▼
인터넷 너머 어딘가에 있는 거대한 컴퓨터 창고
예전에는 회사가 직접 서버(컴퓨터)를 사서 방 한 칸을 채워야 했습니다. 그런데 이걸 매달 요금을 내고 빌려 쓰는 것이 클라우드입니다.

📦 비유: 창고를 직접 짓는 대신 → 창고 임대 서비스를 이용하는 것 필요할 때 크게, 한산할 때 작게 조절할 수 있어 비용과 유연성 모두 이득입니다. AWS(아마존), Azure(마이크로소프트), GCP(구글)이 세계 3대 클라우드입니다.
AWSAzureGCP
🖥️ 온프레미스 (On-Premise) ON-PREM ▼
회사 건물 안에 직접 서버를 두는 방식
클라우드의 반대 개념입니다. 회사가 직접 서버 장비를 구매하고 유지·관리합니다.

🏠 비유: 집을 직접 사서 관리하는 것 ↔ 월세(클라우드) 초기 비용이 크지만 데이터가 회사 밖으로 나가지 않아 보안에 민감한 금융·공공기관이 선호합니다. 메타넷은 온프레미스와 클라우드를 동시에 연결하는 하이브리드 방식 전문 기업입니다.
🔀 하이브리드 클라우드 HYBRID CLOUD ▼
회사 내부 서버 + 클라우드를 함께 사용하는 방식
중요한 데이터는 회사 내부에, 트래픽이 몰리거나 유연성이 필요한 부분은 클라우드에 올리는 혼합 방식입니다.

🍱 비유: 집밥(온프레미스) + 배달음식(클라우드)을 상황에 따라 선택 보안과 유연성을 동시에 챙길 수 있어 대기업·금융기관에서 많이 사용합니다. 메타넷 티플랫폼의 핵심 사업 영역입니다.
🌐 퍼블릭 클라우드 PUBLIC CLOUD ▼
누구나 돈 내고 빌려 쓸 수 있는 클라우드
AWS, Azure, GCP처럼 여러 회사가 같은 클라우드 인프라를 공유해서 사용하는 방식입니다. (물론 데이터는 분리됩니다)

🏢 비유: 고시원이나 셰어하우스 — 공간은 공유하지만 방은 내 것 초기 투자 없이 바로 사용 가능하고, 필요한 만큼만 비용을 냅니다.
🔒 프라이빗 클라우드 PRIVATE CLOUD ▼
한 회사만을 위한 전용 클라우드
회사 내부 또는 전용 데이터센터에 구축하여 오직 그 회사만 사용하는 클라우드입니다.

🏰 비유: 단독 주택 — 아무도 공유 안 하고 내 맘대로 관리 보안이 강하고 커스터마이징이 자유롭지만, 구축 비용이 많이 듭니다. 은행, 정부기관에서 주로 사용합니다.
🗂️ 멀티 클라우드 MULTI CLOUD ▼
AWS, Azure, GCP 등 여러 클라우드를 동시에 쓰는 방식
하나의 회사가 AWS, Azure, 네이버 클라우드를 동시에 사용하는 전략입니다.

🛒 비유: 쿠팡, 11번가, 네이버쇼핑을 동시에 비교하며 최저가 주문 특정 클라우드에 의존하지 않아 장애 위험이 줄고, 각 클라우드의 장점을 골라 쓸 수 있습니다.
🚚 클라우드 마이그레이션 MIGRATION ▼
기존 IT 시스템을 클라우드로 이사시키는 작업
회사가 오래 쓰던 서버와 시스템을 클라우드로 옮기는 것입니다. 그냥 통째로 옮기거나(Lift & Shift), 이사하는 김에 현대식으로 재설계하는 방법 등 여러 전략이 있습니다.

🏚️→🏠 비유: 낡은 집에서 새 아파트로 이사하는 것 메타넷은 NCP/VMware → AWS 같은 복잡한 이사 작업을 전문으로 수행합니다.

📋 IaaS / PaaS / SaaS SERVICE MODELS ▼
클라우드 서비스의 3가지 제공 방식
IaaS (Infrastructure as a Service): 서버·네트워크·저장소 같은 하드웨어만 빌려줌. 건물 뼈대만 제공
PaaS (Platform as a Service): 개발 환경까지 빌려줌. 인테리어까지 된 공간 제공
SaaS (Software as a Service): 완성된 소프트웨어 자체를 빌려줌. 완전히 꾸며진 집에 그냥 입주

📌 예시: IaaS=AWS EC2, PaaS=Google App Engine, SaaS=Gmail·Netflix
⚙️
CHAPTER B MSP & 클라우드 운영
🏭 MSP (Managed Service Provider) MSP ▼
클라우드를 대신 관리해주는 전문 회사
기업이 클라우드를 구축하고 운영하는 전 과정을 대행해주는 IT 서비스 회사입니다. 24시간 365일 모니터링, 비용 최적화, 장애 대응까지 맡아줍니다.

🧑‍🔧 비유: 건물 관리인 — 집주인(기업) 대신 건물(클라우드)의 모든 것을 관리 메타넷은 AWS Advanced Tier MSP 파트너로, 삼성전자·KB국민은행 등 대기업의 클라우드를 관리합니다.
💰 FinOps (클라우드 비용 최적화) FINOPS ▼
클라우드 돈을 낭비 없이 쓰는 관리 방법
클라우드는 쓴 만큼 요금이 나오기 때문에 방심하면 요금폭탄을 맞습니다. FinOps는 클라우드 비용을 실시간으로 분석해 불필요한 낭비를 찾아내고 절감하는 관리 체계입니다.

💡 비유: 전기·가스 요금이 갑자기 늘었을 때 어디서 낭비되는지 찾아 줄이는 것 사용하지 않는 서버 끄기, 저렴한 요금제로 전환 등을 체계적으로 관리합니다.
🔃 DevOps DEVOPS ▼
개발팀과 운영팀이 한 팀처럼 움직이는 방식
Development(개발) + Operations(운영)의 합성어입니다. 원래는 개발팀이 코드 짜고 → 운영팀에 던지는 식이었는데, 이 두 팀이 긴밀하게 협력해 빠르고 안정적으로 소프트웨어를 배포하는 문화·방식입니다.

🏭 비유: 요리사(개발)와 홀 직원(운영)이 함께 메뉴 개선을 논의하는 레스토랑 CI/CD, 자동화 테스트 등의 도구를 사용합니다.
CI/CDGitLabJenkins
🔄 CI/CD (지속적 통합/배포) CI/CD ▼
코드 변경사항을 자동으로 테스트하고 배포하는 파이프라인
CI (Continuous Integration): 개발자들이 코드를 합칠 때마다 자동으로 테스트를 돌려 오류를 즉시 발견
CD (Continuous Delivery/Deployment): 테스트를 통과한 코드를 자동으로 서비스에 배포

🏃 비유: 음식 주문 → 자동 조리 → 자동 배달까지 전 과정 자동화 메타넷은 GitLab, Jenkins, AWS CodeDeploy를 활용해 배포 자동화를 구현합니다.
📜 IaC (Infrastructure as Code) IaC ▼
서버 인프라 설정을 코드로 작성해 자동으로 만드는 방법
서버 100대를 손으로 세팅하는 대신, 코드 한 줄로 자동으로 만들어내는 방식입니다. Terraform, Ansible이 대표 도구입니다.

🖨️ 비유: 집 설계도(코드)만 있으면 버튼 하나로 집 1000채를 자동 건설 실수가 줄고, 동일한 환경을 빠르게 복제할 수 있습니다. 메타넷은 Terraform·Ansible을 주력으로 사용합니다.
TerraformAnsible
📦 컨테이너 (Container) CONTAINER ▼
앱을 독립된 작은 박스에 넣어 어디서나 똑같이 실행시키는 기술
"내 컴퓨터에서는 되는데 서버에선 안 돼요" 문제를 해결합니다. 앱 실행에 필요한 모든 것(코드, 설정, 라이브러리)을 하나의 박스(컨테이너)에 담아, 어떤 환경에서도 동일하게 실행됩니다.

🚢 비유: 국제 표준 컨테이너 — 배에도, 트럭에도, 기차에도 올릴 수 있는 표준 박스 Docker가 대표적인 컨테이너 기술입니다.
DockerKubernetesOpenShift
🎛️ Kubernetes (쿠버네티스) K8S ▼
수천 개의 컨테이너를 자동으로 관리·배분하는 시스템
컨테이너가 수백~수천 개가 되면 혼자 관리하기 불가능합니다. Kubernetes는 컨테이너들을 자동으로 배분하고, 하나가 죽으면 새걸 띄워주고, 트래픽이 몰리면 수를 늘리는 자동 관리 시스템입니다.

🕹️ 비유: 수천 명의 직원(컨테이너)에게 자동으로 업무를 배분하는 AI 인사팀장 구글이 만들었으며, "K8s"(K와 s 사이 8글자)라고도 부릅니다.
🏗️ MSA (마이크로서비스 아키텍처) MSA ▼
하나의 거대한 앱을 작은 기능 단위로 쪼개는 설계 방식
예전에는 쇼핑몰의 주문·결제·배송이 하나의 거대한 프로그램 안에 뭉쳐 있었습니다(모놀리식). MSA는 이를 주문 서비스, 결제 서비스, 배송 서비스처럼 독립적인 작은 단위로 분리합니다.

🍕 비유: 모든 요리를 혼자 하는 식당 → 주방이 역할별로 분리된 공장형 주방 하나가 고장나도 나머지는 정상 동작하고, 각 서비스를 독립적으로 업데이트할 수 있습니다.
🔁 재해복구 DR (Disaster Recovery) DR ▼
지진·화재·해킹 등 재난 발생 시 빠르게 서비스를 복구하는 체계
서버실에 불이 나거나 해킹을 당해도 서비스가 멈추지 않도록 미리 준비하는 계획입니다.

RTO (Recovery Time Objective): 복구에 걸리는 최대 허용 시간 (예: 20분 이내 복구)
RPO (Recovery Point Objective): 데이터를 어느 시점까지 복구할 수 있는가 (예: 10분 전 데이터)

🚑 비유: 화재보험 + 소방 대피 훈련 + 백업 창고를 동시에 갖추는 것 금융기관은 법적으로 DR 센터 구축이 의무입니다. 메타넷은 AWS DRS를 활용해 비용 효율적인 DR을 구축합니다.
AWS DRSRTORPOActive/Active
📊 모니터링 (Monitoring) MONITORING ▼
서버·앱의 상태를 실시간으로 지켜보는 것
CPU 사용률, 응답 속도, 오류 발생 등을 24시간 감시해 문제가 생기면 즉시 알림을 보냅니다.

🏥 비유: 환자의 심박수·혈압을 24시간 체크하는 의료 모니터 메타넷이 사용하는 도구: Datadog, Dynatrace, onTune, Watap, Amazon CloudWatch 등.
DatadogDynatraceCloudWatchObservability
🖥️ 가상화 (Virtualization) VM ▼
물리 서버 1대를 여러 대처럼 사용하는 기술
실제 서버(물리 서버) 1대 위에 소프트웨어를 이용해 여러 개의 가상 컴퓨터(VM)를 만드는 기술입니다.

🏢 비유: 큰 사무실 하나를 파티션으로 나눠 여러 회사가 사용하는 것 서버 자원을 효율적으로 쓸 수 있고, VM 단위로 켜고 끄는 게 가능합니다. VMware vSphere, IBM PowerVM, Red Hat Virtualization이 대표 제품입니다.
VMwareIBM PowerVMHyper-V

CHAPTER C 보안 핵심 개념
🏰 SOC (보안관제센터) SOC ▼
회사의 사이버 보안을 24시간 지키는 전문 팀·센터
Security Operation Center의 약자입니다. 해킹 시도, 악성코드 유입, 이상한 접속 등을 24시간 365일 감시하고, 위협이 발견되면 즉시 대응합니다.

🚨 비유: 경찰서 상황실 — 도시 전체의 CCTV를 모니터링하다 범죄 발생 시 즉시 출동 메타넷의 SOC는 MCSOC(Metanet Cloud Security Operation Center)라 불리며, Splunk 등 최첨단 도구로 운영됩니다.
🤖 SOAR (보안 자동화 대응) SOAR ▼
보안 위협을 자동으로 탐지하고 처리하는 시스템
Security Orchestration, Automation and Response의 약자입니다. 사람이 일일이 처리하던 보안 대응을 자동화해줍니다.

예시: 해킹 의심 IP 탐지 → 자동으로 접속 차단 → 담당자에게 알림 → 보고서 자동 작성 사람이 24시간 붙어 있지 않아도 위협에 빠르게 대응할 수 있습니다. AWS에서는 Lambda 함수로 이를 구현합니다.
자동화LambdaPlaybook
📡 SIEM (보안 정보 이벤트 관리) SIEM ▼
회사 전체의 보안 로그를 한 곳에 모아 분석하는 시스템
Security Information and Event Management의 약자입니다. 서버, 방화벽, PC, 클라우드 등 수천 개 장비에서 발생하는 로그(기록)를 한 곳에 모아 패턴을 분석해 공격을 탐지합니다.

🔬 비유: 수천 명의 통화 기록을 AI로 분석해 사기 전화를 찾아내는 것 메타넷이 사용하는 SIEM: Splunk, Microsoft Sentinel, Netwitness, Logpresso.
SplunkMicrosoft SentinelLogpresso
🧱 방화벽 (Firewall) FIREWALL ▼
허가받지 않은 인터넷 접속을 막는 첫 번째 방어선
회사 네트워크로 들어오고 나가는 모든 데이터를 검사해 허가된 것만 통과시키는 장치·소프트웨어입니다.

🚧 비유: 아파트 입구의 경비원 — 주민증 확인 후 통과, 모르는 사람은 차단 메타넷이 다루는 방화벽: Fortinet, AWS Network Firewall, Azure Firewall 등.
FortinetAWS WAFAzure Firewall
🌐 WAF (웹 방화벽) WAF ▼
웹사이트를 노리는 해킹 공격을 막는 전용 방화벽
Web Application Firewall의 약자입니다. 일반 방화벽이 네트워크 수준을 막는다면, WAF는 웹사이트를 겨냥한 공격(SQL 인젝션, XSS 등)을 전문적으로 막습니다.

🎯 비유: 일반 경비원(방화벽) vs. 해킹 전문 보안 요원(WAF) AWS WAF, Azure WAF 같은 클라우드 내장 WAF도 있습니다.
🔍 EDR (엔드포인트 탐지 및 대응) EDR ▼
직원들의 PC·노트북에서 일어나는 악성 행위를 탐지하는 보안 소프트웨어
Endpoint Detection and Response의 약자입니다. 기존 백신이 알려진 바이러스만 잡는다면, EDR은 행동 패턴을 분석해 신종 악성코드, 랜섬웨어도 탐지합니다.

🕵️ 비유: 도둑의 얼굴을 몰라도 "수상한 행동"을 보고 잡는 AI 경비원 메타넷이 다루는 EDR: CrowdStrike, Cybereason, Microsoft Defender for Endpoint (MDE).
CrowdStrikeCybereasonMDE
🔎 XDR (확장형 탐지 및 대응) XDR ▼
EDR을 넘어 네트워크·클라우드·이메일까지 통합 탐지하는 시스템
Extended Detection and Response의 약자입니다. EDR이 PC만 본다면, XDR은 네트워크, 클라우드, 이메일, 서버까지 전방위로 위협을 탐지하고 상관관계를 분석합니다.

🔭 비유: 동네 경비원(EDR) → 도시 전체를 보는 통합관제센터(XDR) Cortex XDR(Paloalto), Trellix XDR 등이 있습니다.
Cortex XDRTrellixNDR
☁️ CSPM (클라우드 형상 관리) CSPM ▼
클라우드 설정이 잘못되어 생기는 보안 구멍을 찾아주는 도구
Cloud Security Posture Management의 약자입니다. 클라우드를 설정하다 실수로 "모든 사람이 접근 가능"하게 열어두는 경우가 많습니다. CSPM은 이런 잘못된 설정(Misconfiguration)을 자동으로 찾아 수정을 권고합니다.

🔧 비유: 집 문단속 체크리스트 — 창문은 잠겼는지, 가스는 잠겼는지 자동 점검 메타넷은 FRIIM, AWS Security Hub, Microsoft Defender for Cloud를 활용합니다.
FRIIMAWS Security HubPrisma Cloud
🛡️ CWPP (클라우드 워크로드 보안) CWPP ▼
클라우드에서 실행 중인 서버·컨테이너를 직접 보호하는 기술
Cloud Workload Protection Platform의 약자입니다. CSPM이 "설정이 올바른지" 확인한다면, CWPP는 실제로 돌아가고 있는 서버와 컨테이너 내부를 보호합니다.

💊 비유: CSPM = 집 잠금장치 점검 / CWPP = 집 안에서 생활하는 사람 보호 취약점 스캔, 악성코드 탐지, 런타임 보호 등의 기능을 제공합니다.
🚪 CASB (클라우드 접근 보안 중계) CASB ▼
클라우드 서비스 접속을 중간에서 검사·통제하는 보안 게이트
Cloud Access Security Broker의 약자입니다. 직원이 회사 몰래 개인 Dropbox, 구글 드라이브 등에 회사 데이터를 올리는 것을 막고, 승인된 클라우드 서비스만 사용하도록 통제합니다.

🚦 비유: 회사 인터넷을 나가는 모든 데이터의 세관 검사관 메타넷은 Zscaler를 주력 CASB 솔루션으로 활용합니다.
ZscalerShadow IT 차단
🌐 Zscaler (제로 트러스트 게이트웨이) ZTNA ▼
아무도 믿지 말고 항상 검증하는 클라우드 보안 서비스
Zscaler는 Zero Trust(제로 트러스트) 개념 기반의 클라우드 보안 서비스입니다. 회사 네트워크 안에 있다고 해서 다 믿지 말고, 매번 신원을 확인하는 방식입니다.

🛂 비유: 공항 탑승구 — 공항 안에 있어도 탑승권 없으면 비행기 못 탐 재택근무, 외부 접속이 많아진 요즘 필수 보안 솔루션입니다.
🔐 MFA (다단계 인증) MFA / OTP ▼
비밀번호 외에 추가 인증을 요구하는 보안 장치
Multi-Factor Authentication의 약자입니다. 비밀번호(알고 있는 것) + 스마트폰 인증(갖고 있는 것)처럼 두 가지 이상의 방식으로 신원을 확인합니다.

🔑 비유: 카드 + 비밀번호를 함께 요구하는 ATM OTP(일회용 비밀번호)가 대표적인 MFA 방식입니다. 비밀번호가 유출되어도 계정을 지킬 수 있습니다.
RSA SecurIDOTPZero Trust
🏹 APT (지능형 지속 위협) APT ▼
특정 목표를 집중 공략하는 고도로 정교한 해킹 공격
Advanced Persistent Threat의 약자입니다. 국가기관이나 대기업 등 특정 표적을 몇 달~몇 년에 걸쳐 지속적으로 공략하는 고도화된 사이버 공격입니다.

🎯 비유: 일반 소매치기(일반 해킹) vs. 목표 저택만 노리는 전문 강도단(APT) 메타넷은 FireEye 등을 활용해 네트워크·이메일·엔드포인트에서 APT를 탐지합니다.
FireEyeThreat Hunting포렌식
🔬 취약점 진단 (Prowler) VULNERABILITY ▼
클라우드에서 해킹당할 수 있는 구멍을 미리 찾아내는 검사
Prowler는 AWS 환경의 보안 설정을 자동으로 점검하는 오픈소스 도구입니다. 수백 가지 Best Practice 기준으로 잘못된 설정, 보안 허점을 찾아냅니다.

🏥 비유: 건강검진 — 겉으로는 멀쩡해 보여도 내부의 이상을 미리 발견 메타넷은 매월 Prowler 취약점 진단 리포트를 고객사에 제공합니다.
📋 컴플라이언스 (Compliance) COMPLIANCE ▼
법·규정·기준을 지키고 있는지 확인하는 것
금융기관·병원·공공기관은 개인정보보호법, 전자금융감독규정 등 수많은 법적 기준을 지켜야 합니다. 컴플라이언스는 이 기준들을 충족하는지 확인하고 관리하는 활동입니다.

📝 비유: 식당의 위생 검사 — 규정을 지키지 않으면 영업 정지 CSPM 도구가 클라우드 컴플라이언스 자동 점검을 도와줍니다.
🕵️ 포렌식 (Forensics) FORENSICS ▼
사이버 범죄의 증거를 수집하고 분석하는 기술
해킹 사고가 발생했을 때 "누가, 언제, 어떻게, 무엇을 했는지"를 디지털 흔적을 통해 밝혀내는 조사 기술입니다.

🔍 비유: CSI 범죄 수사 — 범행 현장의 지문, 발자국, CCTV 영상을 분석하는 것 메타넷은 사고 발생 시 Lambda 자동화로 포렌식 정보를 즉시 수집합니다.

CHAPTER D AWS 주요 서비스 해설
🟠 AWS (Amazon Web Services) AWS ▼
아마존이 운영하는 세계 1위 클라우드 서비스
전 세계 클라우드 시장의 약 33%를 점유하는 압도적 1위 클라우드 플랫폼입니다. 서버, 데이터베이스, AI, 보안 등 200개 이상의 서비스를 제공합니다.

🌍 넷플릭스, 삼성, 카카오, 금융감독원 등 전 세계 수백만 기업이 사용 메타넷은 AWS Advanced Tier Services Partner 자격을 2025년에 획득했습니다.
⚡ AWS Lambda LAMBDA ▼
서버 없이 코드만 올리면 자동 실행되는 AWS 서비스
서버를 직접 관리하지 않고, 특정 이벤트가 발생할 때만 코드가 실행됩니다(Serverless). 비용도 실행 시간만큼만 냅니다.

⚙️ 보안 자동화 예시: 해킹 탐지 → Lambda 자동 실행 → 위험 IP 즉시 차단 → 담당자 알림 메타넷은 보안 SOAR 자동화에 Lambda를 핵심적으로 활용합니다.
👮 AWS GuardDuty GUARDDUTY ▼
AWS 계정과 리소스를 자동으로 감시하는 위협 탐지 서비스
머신러닝(AI)을 사용해 AWS 환경에서 비정상적인 행동을 탐지합니다. 예를 들어 해외에서 갑자기 대량 데이터 다운로드, 악성 IP에서의 접속 등을 자동으로 발견합니다.

🚨 비유: 은행 계좌에서 이상한 해외 결제가 감지되면 자동으로 카드 정지하는 것
🏢 AWS Security Hub SECURITY HUB ▼
AWS 보안 상태를 한눈에 볼 수 있는 종합 대시보드
GuardDuty, Inspector, Macie 등 여러 AWS 보안 서비스의 결과를 한 곳에 모아 정리해줍니다. 보안 점수를 매겨 어떤 부분이 취약한지 알려줍니다.

📊 비유: 건강검진 종합 결과지 — 각 항목의 점수를 한 페이지에 정리
🌊 Amazon Kinesis Data Firehose KINESIS ▼
대용량 로그·데이터를 실시간으로 수집·전달하는 서비스
초당 수백만 건의 로그 데이터를 실시간으로 수집해 S3(저장소)나 Splunk(SIEM)로 자동 전달합니다.

🔌 비유: 수천 개의 수도꼭지(로그 발생원)를 하나의 대형 파이프로 연결해 저수지(저장소)로 보내는 것 메타넷의 AWS 보안관제 아키텍처에서 핵심 로그 수집 역할을 담당합니다.
📈 Amazon QuickSight QUICKSIGHT ▼
데이터를 시각화하는 AWS 대시보드·BI 서비스
복잡한 보안 데이터·클라우드 비용 데이터 등을 그래프, 차트, 대시보드로 보기 좋게 시각화합니다.

📊 비유: 엑셀 데이터를 예쁜 PPT 그래프로 자동 변환해주는 서비스 메타넷은 QuickSight로 보안 주간·월간 통계 보고서를 자동 생성해 고객사에 제공합니다.
🗒️ AWS CloudTrail CLOUDTRAIL ▼
AWS 계정에서 일어난 모든 행동을 기록하는 로그 서비스
누가, 언제, 어디서, 무엇을 했는지 모든 활동을 기록합니다.

📹 비유: 회사 출입문의 CCTV + 출입 기록부 — 누가 몇 시에 어느 방에 들어갔는지 모두 기록 해킹 사고 조사(포렌식)와 컴플라이언스 감사에 필수적입니다.
🔁 AWS DRS (재해복구 서비스) DRS ▼
장애 발생 시 AWS로 빠르게 복구하는 자동화 DR 서비스
AWS Elastic Disaster Recovery Service의 약자입니다. 온프레미스 서버나 다른 클라우드의 데이터를 AWS로 지속적으로 복제해두고, 재해 발생 시 버튼 하나로 즉시 복구합니다.

💾 비유: 게임 자동저장 — 갑자기 전원이 나가도 마지막 저장 시점으로 즉시 복구 RPO 10분 이하, RTO 20분 이하 수준의 빠른 복구가 가능합니다.
🌐 VPC (가상 사설 네트워크) VPC ▼
AWS 클라우드 안에 내 전용 가상 네트워크를 만드는 것
Virtual Private Cloud의 약자입니다. AWS 안에 내 회사만의 독립된 네트워크 공간을 만들어, 다른 고객의 리소스와 완전히 분리합니다.

🏘️ 비유: 아파트 단지 안에 담장으로 분리된 내 집만의 마당 메타넷은 멀티 VPC 아키텍처를 설계해 보안/개발/운영 환경을 격리합니다.
🔗 AWS Direct Connect DX ▼
회사 IDC와 AWS를 전용선으로 직접 연결하는 서비스
일반 인터넷 대신 물리적 전용 회선으로 AWS와 직접 연결합니다. 인터넷보다 빠르고, 안정적이며, 보안도 강합니다.

🛣️ 비유: 일반 도로(인터넷) 대신 회사 전용 고속도로를 AWS까지 직접 연결 금융기관처럼 대용량 데이터를 안정적으로 처리해야 하는 곳에서 필수입니다.
🚀 Auto Scaling AUTO SCALING ▼
트래픽에 따라 서버 수를 자동으로 늘리거나 줄이는 기능
명절에 쇼핑몰 접속자가 10배 몰려도 서버가 자동으로 늘어나고, 한산할 때는 줄어들어 비용을 절감합니다.

🚕 비유: 출퇴근 시간에 버스를 더 투입하고, 한산할 때 줄이는 것 과거에는 최대치 기준으로 서버를 사서 낭비했지만, Auto Scaling으로 필요한 만큼만 사용합니다.
🔎 ECIRA (AWS 차세대 보안관제) ECIRA ▼
AWS가 공식 인정한 차세대 클라우드 보안관제 플랫폼
Enhanced Cloud Incident Response Assessment의 약자입니다. AWS 환경에 최적화된 보안관제 체계로, 침해 탐지→분석→대응의 전 과정을 통합 지원합니다.

🏅 메타넷은 ECIRA를 활용한 국내 1호 파트너로, 2023년 AWS로부터 공식 인정을 받았습니다. Prowler 진단 + GuardDuty + Lambda 자동화를 결합한 최첨단 보안 체계입니다.

CHAPTER E 인프라 & 네트워크 기술
🗄️ 서버 (Server) SERVER ▼
다른 컴퓨터에게 서비스를 제공하는 강력한 컴퓨터
여러분이 카카오톡을 보낼 때, 그 메시지를 받아서 상대방에게 전달하는 것이 서버입니다. 24시간 꺼지지 않고 수천~수백만 명의 요청을 처리합니다.

🍽️ 비유: 레스토랑의 주방 — 손님(사용자)의 주문(요청)을 처리해 서빙 메타넷이 다루는 서버 브랜드: IBM, Dell, HPE, Lenovo, Cisco, Nutanix.
💾 스토리지 (Storage) STORAGE ▼
기업의 방대한 데이터를 저장하는 대용량 저장장치
기업용 스토리지는 개인 USB나 하드디스크와는 차원이 다릅니다. 페타바이트(1,000TB) 단위의 데이터를 빠르고 안전하게 저장합니다.

📦 비유: 개인 사물함(개인 저장장치) vs. 대형 물류 창고(기업용 스토리지) IBM Flash Storage, Dell EMC, NetApp 등이 대표적입니다.
🏗️ HPC (고성능 컴퓨팅) HPC ▼
기상예측·AI 학습·시뮬레이션 등 엄청난 계산을 처리하는 컴퓨팅
High Performance Computing의 약자입니다. 수천 개의 CPU와 GPU를 연결해 일반 컴퓨터로는 수백 년 걸릴 계산을 몇 시간 만에 처리합니다.

🚀 비유: 일반 자동차(개인 PC) vs. F1 레이싱카(HPC) 메타넷은 삼성전자 DS부문의 반도체 시뮬레이션용 HPC를 AWS/Azure 멀티 클라우드로 운영합니다.
🏛️ 데이터센터 (IDC) IDC / DC ▼
수천~수만 대의 서버가 모여 있는 전용 건물
인터넷 Data Center의 약자입니다. 냉방, 전력 이중화, 소화 설비, 물리 보안 등을 갖춘 서버 전용 건물입니다.

🏭 비유: 서버들의 아파트 단지 — 24시간 전기, 냉방, 보안이 완비된 건물 클라우드의 등장으로 기업들이 IDC를 직접 운영하는 대신 클라우드를 빌려 쓰는 추세입니다.
🔴 Red Hat OpenShift OPENSHIFT ▼
기업용 컨테이너 관리 플랫폼 (Kubernetes 기반)
Kubernetes를 기업 환경에 맞게 더 사용하기 쉽고 안전하게 포장한 Red Hat의 플랫폼입니다. 보안, 모니터링, CI/CD가 내장되어 있습니다.

🚗 비유: Kubernetes = 자동차 엔진, OpenShift = 엔진에 에어백·네비·편의기능을 더한 완성차 메타넷은 금융기관·대기업의 OpenShift 구축을 전문으로 수행합니다.
💠 VMware vSphere / vCenter VMWARE ▼
서버 가상화의 업계 표준 소프트웨어
VMware는 물리 서버 1대 위에 수십 개의 가상 서버를 만들 수 있는 가상화 소프트웨어입니다.

vSphere: 가상화 엔진 (서버 위에 VM을 만드는 핵심 소프트웨어)
vCenter: 여러 VMware 서버를 한 화면에서 통합 관리하는 도구
vSAN: 여러 서버의 하드디스크를 하나로 묶어 가상 스토리지를 만드는 기능

🏢 비유: 큰 사무실(물리 서버)을 vSphere로 여러 칸으로 나누고, vCenter로 전 건물을 관리
🔧 Ansible (앤서블) ANSIBLE ▼
수천 대 서버 설정을 코드 한 줄로 자동화하는 도구
Red Hat이 만든 IT 자동화 도구입니다. 서버 1,000대에 새 소프트웨어 설치·설정을 코드 한 번 실행으로 동시에 처리합니다.

🤖 비유: 공장 자동화 로봇 — 사람이 일일이 할 작업을 자동으로 처리 YAML 파일로 원하는 상태를 선언하면 Ansible이 알아서 맞춰줍니다.
🌍 Terraform TERRAFORM ▼
AWS·Azure 등 클라우드 인프라를 코드로 관리하는 도구
HashiCorp가 만든 IaC(인프라 자동화) 도구입니다. AWS, Azure, GCP 등 여러 클라우드를 하나의 코드로 통합 관리할 수 있습니다.

📐 비유: 건축 설계도(Terraform 코드) → 시공사(클라우드)에 던지면 자동 건설 "선언형" 방식 — "서버 10대 필요해" 라고 쓰면, Terraform이 알아서 만들어줍니다.
💿 백업 (Backup) & Veeam BACKUP ▼
데이터를 잃어버려도 복구할 수 있도록 미리 복사해두는 것
랜섬웨어(데이터 암호화 후 돈 요구), 실수 삭제, 재해 등에 대비해 데이터를 다른 곳에 주기적으로 복사해두는 작업입니다.

📸 비유: 스마트폰 자동 사진 백업 — 폰을 잃어버려도 사진은 살아있는 것 Veeam은 기업용 백업·복구·아카이빙 소프트웨어의 글로벌 1위 브랜드입니다.

🔄 CDC (데이터 변경 포착) CDC ▼
데이터베이스의 변경사항을 실시간으로 다른 곳에 전달하는 기술
Change Data Capture의 약자입니다. 데이터베이스에서 새로 추가되거나 수정·삭제된 데이터를 실시간으로 감지해 다른 시스템(데이터 웨어하우스, 클라우드 등)에 반영합니다.

📡 비유: 원본 문서의 변경사항을 실시간으로 여러 사람에게 공유하는 Google Docs 메타넷은 Quest의 CDC 솔루션을 활용합니다.

CHAPTER F 주요 보안 도구 & 플랫폼 해설
🟣 Splunk SPLUNK ▼
대용량 로그를 실시간 분석하는 세계 1위 SIEM 플랫폼
하루에 테라바이트 단위로 쌓이는 서버 로그, 보안 이벤트를 실시간으로 검색·분석·시각화합니다. 머신러닝으로 이상 패턴도 자동 탐지합니다.

🔍 비유: 수억 개의 문자메시지 중에서 사기 문자를 1초 만에 찾아내는 AI 메타넷은 Splunk Gold Standard Level 운영 체계를 갖춘 국내 최고 수준의 Splunk 파트너입니다.
🔵 Microsoft Sentinel SENTINEL ▼
마이크로소프트의 클라우드 네이티브 SIEM + SOAR 통합 솔루션
Azure 위에서 동작하는 클라우드 SIEM입니다. 별도 서버 구축 없이 Microsoft 365, Azure, 서드파티 도구의 로그를 바로 연동해 분석합니다.

☁️ 기존 SIEM은 서버를 직접 구축해야 했지만, Sentinel은 클라우드에서 바로 시작 가능 Logic App으로 자동화 대응(SOAR)도 내장되어 있습니다.
🐾 CrowdStrike Falcon CROWDSTRIKE ▼
클라우드 기반의 AI 엔드포인트 보안 플랫폼
직원 PC·서버에 가벼운 에이전트를 설치하면, 모든 행동 데이터가 CrowdStrike 클라우드로 전송되어 AI가 실시간으로 위협을 분석합니다.

🦅 "매(Falcon)"처럼 위협을 빠르게 발견해 잡는다는 의미의 브랜드명 랜섬웨어, 파일리스 공격, APT 등 최신 위협에 특히 강합니다.
🟢 Datadog DATADOG ▼
클라우드 인프라·앱·보안을 통합 모니터링하는 플랫폼
서버 성능, 애플리케이션 오류, 클라우드 비용, 보안 이벤트를 하나의 대시보드에서 모두 볼 수 있습니다.

📡 비유: 자동차의 계기판 — 속도·연료·엔진 온도를 한 화면에서 한눈에 파악 메타넷은 2023년 Datadog 'Voice of Partner'로 선정된 공식 파트너입니다.
🔶 Fortinet FORTINET ▼
네트워크 방화벽의 글로벌 1위 보안 회사
FortiGate 방화벽으로 유명한 네트워크 보안 전문 기업입니다. 방화벽, IPS(침입방지), VPN, 웹 필터링을 하나의 장비(FortiGate)에 통합합니다.

🏰 비유: 성벽(방화벽) + 해자 + 경비병 + CCTV가 하나로 합쳐진 통합 방어 시스템 클라우드 환경에서는 FortiGate-VM으로 가상 방화벽도 제공합니다.
🟡 Nutanix (뉴타닉스) HCI ▼
서버+스토리지+네트워크를 하나로 합친 초융합 인프라
HCI(Hyper-Converged Infrastructure)의 대표 브랜드입니다. 예전에는 서버, 스토리지, 네트워크 장비를 따로 사야 했지만, Nutanix는 이를 하나의 박스로 통합합니다.

📱 비유: 카메라+전화+음악플레이어+내비가 하나로 합쳐진 스마트폰 설치와 관리가 매우 간단해지고, 필요에 따라 노드(박스)를 추가해 확장할 수 있습니다.
🔷 OpenSearch (오픈서치) OPENSEARCH ▼
대량의 데이터를 빠르게 검색하고 분석하는 오픈소스 엔진
AWS가 오픈소스로 제공하는 검색·분석 엔진입니다. 메타넷의 AWS 보안관제 아키텍처에서 보안 로그 검색 엔진으로 활용됩니다.

🔍 비유: 수억 개의 로그 중에서 원하는 내용을 구글처럼 빠르게 검색 Elasticsearch의 오픈소스 후계자이며, AWS Kinesis로 수집된 로그를 저장·검색합니다.
🟤 SAP (기업 전사 자원 관리) SAP ERP ▼
회사의 회계·인사·물류·생산을 하나로 통합하는 기업용 소프트웨어
SAP는 독일 기업으로, 대기업에서 쓰는 ERP(Enterprise Resource Planning) 소프트웨어의 세계 1위입니다. 회계, 인사, 구매, 물류, 생산 관리를 하나의 시스템으로 통합합니다.

🏭 비유: 회사 전체 부서의 업무를 하나의 앱으로 연결하는 것 메타넷은 SAP를 클라우드(Azure, AWS)로 이전하는 프로젝트를 다수 수행합니다.
📋
CHAPTER G 프로젝트 관리 용어
📐 아키텍처 (Architecture) ARCHITECTURE ▼
IT 시스템의 전체 설계도
서버, 네트워크, 데이터베이스, 보안 등 IT 시스템의 각 요소가 어떻게 연결되고 동작하는지를 나타낸 전체 그림입니다.

🏗️ 비유: 집 짓기 전의 건축 설계도 — 어디에 방이 오고, 어떻게 연결되는지 미리 계획 좋은 아키텍처는 확장성, 보안, 성능, 비용을 모두 고려합니다. 메타넷의 SA(Solution Architect)가 이를 담당합니다.
🗺️ PMO (프로젝트 관리 조직) PMO ▼
대형 IT 프로젝트를 계획대로 완료시키는 전문 관리 조직
Project Management Office의 약자입니다. 일정, 예산, 인력, 리스크를 체계적으로 관리해 프로젝트가 목표대로 완성되도록 합니다.

🎯 비유: 건물 공사 현장의 현장 감독 — 각 팀이 제때 제대로 하는지 총괄 조율 메타넷 직원의 18.3%가 PMO 역할을 수행하는 전문 인력입니다.
🔍 SA/SE (솔루션 아키텍트 / 시스템 엔지니어) SA / SE ▼
고객 요구사항을 기술적으로 설계하고 구현하는 전문가
SA (Solution Architect): 고객의 비즈니스 요구를 분석해 최적의 IT 시스템 구조를 설계하는 전문가
SE (Systems Engineer): 설계된 시스템을 실제로 구축·테스트하는 기술 전문가

🏠 SA = 건축 설계사, SE = 실제 시공 기술자 메타넷 전체 인력의 46.5%가 SA/SE 업무를 담당합니다.
🤝 SLA (서비스 수준 협약) SLA ▼
서비스 품질 기준을 계약으로 보장하는 약속
Service Level Agreement의 약자입니다. "서버 가동률 99.9% 이상 보장", "장애 발생 시 4시간 이내 복구" 같이 서비스 품질 기준을 수치로 명확히 계약하는 것입니다.

📜 비유: 택배 "내일 배송 보장" 서비스 — 못 지키면 환불해주는 명확한 약속 메타넷은 고객사와 SLA를 맺고 24×365 모니터링으로 이를 준수합니다.
🔢 7R 전략 (클라우드 이전 7가지 방법) 7R ▼
기존 시스템을 클라우드로 옮기는 7가지 전략
AWS가 정의한 클라우드 마이그레이션 전략 프레임워크입니다.

Rehost: 그냥 통째로 옮김 (Lift & Shift)
Replatform: 조금만 수정해서 옮김
Repurchase: SaaS로 교체
Refactor: 클라우드 네이티브로 재설계
Retire: 안 쓰는 것 폐기
Retain: 당분간 유지
Relocate: 다른 클라우드로 이전

🏡 비유: 이사할 때 — 가구를 그대로 가져갈지, 새로 살지, 버릴지 결정하는 것
🏷️ TCO (총 소유 비용) TCO ▼
IT 시스템을 도입해서 운영하는 데 드는 모든 비용의 합계
Total Cost of Ownership의 약자입니다. 장비 구매비만이 아니라 전기료, 인건비, 유지보수비, 업그레이드 비용 등을 모두 합산한 실제 비용입니다.

🚗 비유: 자동차 가격만 보면 안 되고, 기름값+보험+수리비+주차비를 합쳐야 진짜 비용 클라우드로 전환하면 TCO를 30~50% 줄일 수 있다는 것이 일반적인 분석입니다.
